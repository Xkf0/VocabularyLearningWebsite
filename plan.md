# 方案：一键启动，无需手动运行 Python

## 目标
让 "小白" 用户只需双击一个文件就能使用，无需接触终端、Python、localhost 等概念。

## 方案设计

### 核心思路
**双保险：有 Python 用服务器模式，没 Python 用浏览器模式。批处理文件统一入口。**

---

### 1. 创建 `启动.bat`（智能启动器）

双击这个文件，自动判断：

```
启动.bat 执行流程:
  ├─ 检查 Python 是否可用
  │   ├─ 有 Python → 启动 server.py → 等待2秒 → 打开浏览器 http://localhost:3000
  │   │              数据自动保存到 vocabulary-data.json
  │   └─ 无 Python → 打开 index.html（浏览器 localStorage 模式）
  │                  数据保存在浏览器中，可手动导出备份
  └─ 提示用户如果浏览器没自动打开，手动访问对应地址
```

- 终端窗口保持打开状态（有 Python 时），用户关闭终端 = 关闭服务器
- 窗口标题显示「艾宾浩斯背单词 - 服务器运行中，关闭此窗口即停止」

### 2. 优化 `index.html` 的离线体验

让 localStorage 模式对小白用户更友好：

- **修改文件状态栏文案**：不用 "离线模式" 这样有歧义的词，改为更友好的表述
- **新增「保存到文件」按钮**：在设置页加一个醒目的一键备份按钮，下载 `vocabulary-data.json`
- **新增自动备份提醒**：在设置页加选项，每隔一段时间自动提示备份
- **优化设置页说明**：用更通俗的语言，去掉技术术语

### 3. 可选：修改 `server.py` 增加自动打开浏览器

当用户直接运行 `python server.py` 时，自动打开浏览器，减少手动输入 URL 的步骤。

---

## 涉及文件

| 文件 | 操作 |
|------|------|
| `启动.bat` | **新建** - 智能启动器 |
| `index.html` | **修改** - 优化离线模式体验，新增保存到文件功能 |
| `server.py` | **可选修改** - 增加自动打开浏览器功能 |

## 不需要改动的
- 已有的服务器/离线双模式逻辑（已经很好）
- 复习逻辑、间隔算法
- 音标和音频功能
