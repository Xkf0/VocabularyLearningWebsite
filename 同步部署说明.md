## 第一步：设置 SSH 免密码登录（每台电脑只需做一次）

### 1. 生成密钥

打开 PowerShell，执行：

```powershell
ssh-keygen -t rsa -b 2048
```

一路回车（不设密码）。

### 2. 复制公钥到服务器

```powershell
# 先查看公钥内容，选中整行复制
Get-Content $env:USERPROFILE\.ssh\id_rsa.pub
```

SSH 登录服务器，把公钥贴进去：

```powershell
ssh root@YOUR_SERVER_IP
```

输入密码 `YOUR_PASSWORD` 进去后，依次执行：

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
echo "这里粘贴你刚才复制的公钥内容" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
chmod 755 /root
exit
```

### 3. 修复 Windows 私钥权限（新电脑必做！）

Windows OpenSSH 对私钥文件权限非常严格。**每台新电脑**运行完上面两步后，必须执行：

```powershell
# 修复私钥权限（只保留当前用户和 SYSTEM，去掉 Administrators 组）
icacls "$env:USERPROFILE\.ssh\id_rsa" /inheritance:r /grant "$env:USERNAME:(R)" /grant "SYSTEM:(F)"
```

### 4. 验证免密码登录

```powershell
ssh -o BatchMode=yes root@YOUR_SERVER_IP "echo 成功了"
```

如果不问密码直接显示 `成功了`，说明设置成功。

### 5.（服务器只需做一次）关闭 StrictModes

阿里云安全组件会定期把 `/root` 目录权限重置为 777，导致 SSH 拒绝密钥登录。
**在服务器上执行一次即可永久解决**：

```bash
echo "StrictModes no" > /etc/ssh/sshd_config.d/disable-strictmodes.conf
systemctl restart sshd
```

## 第二步：同步代码到服务器

### 方法一：双击 sync_server.bat（推荐）

项目目录下的 `sync_server.bat` 一键同步，会自动：
1. 打包整个项目（排除 .git 和 __pycache__），通过 SSH 管道直接传输到服务器
2. 用 systemctl 重启服务（立即返回，不会卡住）

### 方法二：手动操作

```powershell
# 进入项目目录
cd .

# 打包并上传（SSH 管道传输，不需要中间文件）
tar --exclude=".git" --exclude="__pycache__" --exclude="*.bat" -cf - . | ssh -o BatchMode=yes root@YOUR_SERVER_IP "cd /root && tar -xf -"

# 重启系统服务（systemd 方式，立即返回）
ssh -o BatchMode=yes root@YOUR_SERVER_IP "systemctl restart vocab-server"
```

### 方法三：单独上传某个文件

```powershell
ssh -o BatchMode=yes root@YOUR_SERVER_IP "cat > /root/filename" < .\filename
```

## 疑难解答（逐个排查）

如果你换电脑后 SSH 连不上或同步脚本报错，按顺序排查以下问题：

### Q1: SSH 让我输密码，明明已经配了密钥

**原因**：`/root` 目录权限被阿里云安全组件重置为 777，SSH 拒绝使用密钥登录。

**解决**：

```bash
# 先密码登录进去
ssh root@YOUR_SERVER_IP

# 查看 /root 权限
ls -ld /root
# 如果显示 drwxrwxrwx，说明被重置了，但没关系

# 确保 StrictModes 已关闭（只需做一次）
cat /etc/ssh/sshd_config.d/disable-strictmodes.conf
# 如果文件不存在或输出不对，重新创建：
echo "StrictModes no" > /etc/ssh/sshd_config.d/disable-strictmodes.conf
systemctl restart sshd
```

**原理**：阿里云安全组件会定期把 `/root` 恢复成 777，设置 755 只能临时有用。正确做法是关闭 StrictModes，让它不检查 `/root` 的权限。一劳永逸。

---

### Q2: 双击 sync_server.bat 仍然问密码，但 PowerShell 里 SSH 免密码正常

**原因**：cmd.exe（双击运行 .bat 的程式）用的 SSH 和 PowerShell 不是同一个。双击 .bat 时可能跑到 Git Bash 的 SSH 去，它不认识 Windows 的密钥。

**解决**：确保 `sync_server.bat` 里用的是 Windows 系统自带的 SSH 绝对路径：

```batch
set ssh=C:\Windows\System32\OpenSSH\ssh.exe
```

不要用 `ssh.exe`（不加路径），因为 PATH 环境变量里可能先找到 Git Bash 的 SSH。

---

### Q3: icacls 修复权限时提示"无法枚举"

**原因**：PowerShell 里 `$env:USERNAME:(R)` 的括号被当作命令执行了。

**解决**：用引号包起来，不要从变量拼接：

```powershell
icacls C:\Users\YOUR_USER\.ssh\id_rsa /inheritance:r /grant "YOUR_USER:(R)" /grant "SYSTEM:(F)"
```

---

### Q4: ssh 能连上，但 scp 传文件失败（SSL 错误）

**原因**：Windows 自带的 OpenSSH scp 有兼容性问题。

**解决**：不要用 scp，改用 SSH 管道传文件：

```powershell
# 传单个文件
ssh -o BatchMode=yes root@YOUR_SERVER_IP "cat > /root/filename" < localfile

# 传整个项目
tar --exclude=".git" --exclude="__pycache__" -cf - . | ssh -o BatchMode=yes root@YOUR_SERVER_IP "cd /root && tar -xf -"
```

---

### Q5: 同步后网站打不开（502 / 连接拒绝）

**原因**：可能是服务器 Python 进程挂了，或者根本没启动起来。

**解决**：

```bash
# 1. 查看服务状态
systemctl status vocab-server

# 2. 看最近日志
journalctl -u vocab-server -n 20 --no-pager

# 3. 如果没运行，手动拉起
systemctl restart vocab-server

# 4. 确认端口在监听
ss -tlnp | grep 3000
```

---

### Q6: 旧版本的同步脚本（pkill / nohup / setsid）会卡住或杀进程后网站挂了

**背景**：`pkill -f 'python3 server.py'` 会匹配到 SSH 会话本身，把 SSH 连接也杀掉。`nohup` 和 `setsid` 在非交互 SSH 下经常不生效，进程随 SSH 断开而死亡。

**解决**：使用 systemd 服务管理进程，不要手动启停：

```bash
# 查看 systemd 服务定义
cat /etc/systemd/system/vocab-server.service

# 内容如下：
# [Unit]
# Description=Vocab Learning Server
# After=network.target
#
# [Service]
# ExecStart=/usr/bin/python3 /root/server.py
# WorkingDirectory=/root
# Restart=always
# User=root
#
# [Install]
# WantedBy=multi-user.target
```

同步脚本只需要 `systemctl restart vocab-server`，它会自动杀掉旧进程、启动新进程，然后立即返回。

---

### Q7: .bat 文件执行时中文显示乱码

**原因**：cmd.exe 默认编码是 GBK，而 .bat 文件保存为 UTF-8。

**解决**：在 .bat 第一行加：

```batch
chcp 65001 >nul
```

---

### Q8: .bat 文件里 `http://...` 报错"找不到命令"

**原因**：cmd.exe 把 `://` 的斜杠解析为路径分隔符。

**解决**：用 caret `^` 转义冒号：

```batch
echo 浏览器刷新 http:^//YOUR_SERVER_IP:3000
```

## 首次部署步骤总结（新服务器从头搭建）

如果换了一台全新的阿里云服务器，依次执行：

1. **SSH 密钥配置**（按第一步的 1~4）
2. **关闭 StrictModes**：`echo "StrictModes no" > /etc/ssh/sshd_config.d/disable-strictmodes.conf && systemctl restart sshd`
3. **装 Python**：`apt update && apt install -y python3`
4. **同步代码**：双击 `sync_server.bat`
5. **创建 systemd 服务**（只需一次）：

```bash
cat > /etc/systemd/system/vocab-server.service << 'EOF'
[Unit]
Description=Vocab Learning Server
After=network.target

[Service]
ExecStart=/usr/bin/python3 /root/server.py
WorkingDirectory=/root
Restart=always
User=root

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable vocab-server
systemctl start vocab-server
```

6. **验证**：访问 http://YOUR_SERVER_IP:3000
