# 问题记录

## 电脑能 Ping 通服务器但浏览器卡 "处理中"

**现象**：某台电脑访问词汇学习网站，页面能加载，但点击登录后按钮一直显示"处理中..."，无报错，页面卡死。

**排查过程**：
1. 服务器端口从 3000 改为 80（某些网络封锁非标准端口）→ 无效
2. 浏览器无痕模式 → 无效
3. 换浏览器（Chrome ↔ Edge）→ 无效
4. 控制台 `fetch('/api/auth/verify')` → 正常返回 401（GET 请求通）
5. 控制台 `fetch('/api/auth/login', {method:'POST', ...})` → Promise 一直 pending
6. PowerShell `curl.exe http://114.55.176.249/api/ping` → 秒出（系统层面网络通）
7. PowerShell `Test-NetConnection 114.55.176.249 -Port 80` → `TcpTestSucceeded: True`

**根因**：该电脑开启了 **VPN / 代理（梯子）并且是全局模式**。浏览器流量走了代理，代理到阿里云服务器的 HTTP 请求卡死。

- 系统命令（PowerShell curl / Test-NetConnection）不走代理，所以正常
- GET 请求在某些代理配置下能通但 POST 请求卡住
- 另一台电脑梯子是 PAC 模式，阿里云 IP 被识别为中国地址直连，所以正常

**解决方案**：
- 关闭 VPN/梯子
- 或将梯子改为 PAC 模式（非全局模式）
- 或在梯子的直连列表中添加 `114.55.176.249`
